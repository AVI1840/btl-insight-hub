# פרומפט — מערכת דוחות, ניתוח ושאילתות BTL

---

## הקשר

אתה מהנדס Frontend בכיר ומעצב מוצר. בנה מערכת דוחות, ניתוח ושאילתות אינטראקטיבית עבור ביטוח לאומי — ארגון ציבורי גדול עם מיליוני פניות אזרחים בשנה.

המערכת מיועדת למנהלים בכל הדרגים: מנהלות אגף, סמנכ"לים, ראשי מוקדים, ומנהלי תחום. היא מאפשרת להם לשלוף דוחות, לנתח מגמות, לשאול שאלות על הדאטה, ולקבל התרעות אוטומטיות — הכל מבוסס על שלושה מקורות מידע מרכזיים.

## מה המערכת צריכה לעשות

### 1. דוחות לפי תאריכים
- **תקופות:** יומי, שבועי, חודשי, רבעוני, שנתי, טווח מותאם אישית
- **חתכים:** לפי אגף, לפי מקור (מפ"צ/קש"ב/קש"ר), לפי נושא, לפי חומרה
- **סוגי דוחות:**
  - סיכום מנהלים (Executive Summary) — פסקה בשפה טבעית
  - דוח ביקוש כשל (Failure Demand) — כמה אזרחים חזרו ולמה
  - דוח כשלים שורשיים (Root Cause) — מה שבור ומה ההשפעה
  - דוח סנטימנט — מה האזרחים מרגישים (כעס, תסכול, בלבול)
  - דוח נפח ומגמות — עליות/ירידות בפניות לפי נושא

### 2. ניתוח (Analytics)
- **מגמות לאורך זמן** — האם בעיה מחמירה או משתפרת?
- **השוואה בין תקופות** — אפריל מול מרץ, Q1 מול Q1 שנה שעברה
- **Drill-down** — מאגף → לנושא → לכשל ספציפי → לציטוטי אזרחים
- **זיהוי אנומליות** — קפיצה חריגה בנושא מסוים
- **ניתוח אוכלוסיות** — מי הנפגעים? (גיל, אזור, סוג תביעה)

### 3. שאילתות (Ask the Data)
- ממשק שאילתות בשפה טבעית (עברית)
- דוגמאות:
  - "כמה פניות חוזרות היו באבטלה בחודש האחרון?"
  - "מה הנושא הכי כואב במילואים ברבעון 1?"
  - "הראה לי ציטוטים של אזרחים שהתלוננו על קוד סודי"
  - "מה השתנה בנכות בין ינואר לאפריל?"
- תשובות בשפה טבעית + נתונים תומכים
- אפשרות לשמור שאילתות כ"דוח מהיר"

### 4. התרעות אוטומטיות

#### לפי תדירות:
| תדירות | מה נשלח | למי |
|---------|---------|-----|
| **יומי** | חריגות קריטיות (קפיצה של 20%+ בנושא) | מנהל מוקד, ראש תחום |
| **שבועי** | סיכום שבועי + Top 3 כשלים חדשים | מנהלת אגף |
| **חודשי** | דוח מנהלים מלא + המלצות לפעולה | סמנכ"ל, מנהלת אגף |
| **רבעוני** | דוח אסטרטגי + מגמות + ROI של פעולות | סמנכ"ל, מנכ"ל |
| **שנתי** | סיכום שנתי + תוכנית שיפור | הנהלה בכירה |

#### לפי פוזיציה (Role-Based):
| פוזיציה | מה רואה | רמת פירוט |
|---------|---------|-----------|
| **מנכ"ל / סמנכ"ל** | תמונת מצב ארגונית, מדד כאב, ביקוש כשל, המלצות אסטרטגיות | גבוה מאוד — שורה תחתונה בלבד |
| **מנהלת אגף** | ניתוח מעמיק של האגף שלה, כשלים, מגמות, פעולות מומלצות | בינוני-גבוה |
| **ראש מוקד** | ביצועי מוקד, קודי סגירה בעייתיים, נציגים שצריכים הדרכה | מפורט |
| **מנהל תחום** | Drill-down לנושא ספציפי, אוכלוסיות מושפעות, ציטוטים | מפורט מאוד |

#### סוגי התרעות:
- 🔴 **קריטית** — חריגה חמורה שדורשת תגובה מיידית
- 🟡 **אזהרה** — מגמה מדאיגה שצריך לעקוב אחריה
- 🟢 **עדכון** — שיפור שזוהה או פעולה שהצליחה
- 📊 **דוח תקופתי** — סיכום מתוזמן לפי לוח זמנים

#### ערוצי שליחה:
- דשבורד (in-app notification)
- אימייל מעוצב
- SMS (לקריטיות בלבד)
- Push notification (אפליקציה עתידית)

### פילוסופיית עיצוב
המערכת הזו **לא דשבורד BI קלאסי**. היא כלי עבודה יומי למנהלים עסוקים. העיצוב צריך להיות:

1. **ברור מיידית** — מנהל פותח ותוך 5 שניות מבין מה המצב
2. **לא מציף** — מידע מדורג: שורה תחתונה → פירוט → drill-down
3. **פעיל** — לא רק מראה, מציע מה לעשות
4. **מותאם אישית** — כל פוזיציה רואה מה שרלוונטי לה
5. **נגיש** — עברית מלאה, RTL, ניגודיות גבוהה, גדלי טקסט קריאים

## דרישות טכניות

### סטאק:
- **Frontend:** React 19 + TypeScript + Vite + Tailwind CSS 4
- **Icons:** Lucide React
- **State:** React useState/useReducer (ללא Redux)
- **Data Layer:** Typed mock data ב-`src/data/` — מוכן להחלפה ב-API
- **RTL:** מלא, עברית, כיוון ימין-לשמאל

### מבנה קבצים:
```
src/
├── components/
│   ├── layout/        (Sidebar, Header, PageShell)
│   ├── reports/       (ReportViewer, ReportFilters, ReportExport)
│   ├── analytics/     (TrendCard, ComparisonView, DrillDown)
│   ├── queries/       (QueryInput, QueryResult, SuggestedQueries)
│   ├── alerts/        (AlertList, AlertConfig, AlertBadge)
│   └── shared/        (Card, Badge, Button, ProgressBar, DatePicker)
├── data/
│   ├── reports.ts     (mock report data, types)
│   ├── analytics.ts   (mock trends, comparisons)
│   ├── queries.ts     (mock Q&A pairs)
│   └── alerts.ts      (mock alerts, config types)
├── hooks/
│   ├── useFilters.ts
│   ├── useAlerts.ts
│   └── useQuery.ts
├── types/
│   └── index.ts       (shared types: Domain, Period, Role, etc.)
├── App.tsx
├── main.tsx
└── index.css
```

### Types מרכזיים:
```typescript
type Domain = 'אבטלה' | 'מילואים' | 'נכות' | 'ילדים' | 'זיקנה' | 'שיקום' | 'קצבאות'
type Source = 'מפ"צ' | 'קש"ב' | 'קש"ר'
type Period = 'daily' | 'weekly' | 'monthly' | 'quarterly' | 'yearly' | 'custom'
type Role = 'ceo' | 'vp' | 'dept_manager' | 'call_center_manager' | 'team_lead'
type AlertSeverity = 'critical' | 'warning' | 'info' | 'positive'
type AlertChannel = 'dashboard' | 'email' | 'sms' | 'push'

interface Report {
  id: string
  title: string
  period: Period
  startDate: string
  endDate: string
  domain: Domain | 'all'
  source: Source | 'all'
  summary: string           // AI-generated narrative
  insights: ReportInsight[]
  rootCauses: RootCause[]
  affectedPopulations: Population[]
  recommendations: Recommendation[]
  generatedAt: string
}

interface AlertConfig {
  id: string
  role: Role
  domains: Domain[]
  frequency: Period
  channels: AlertChannel[]
  threshold: number         // % change that triggers alert
  enabled: boolean
}

interface Alert {
  id: string
  severity: AlertSeverity
  title: string
  description: string
  domain: Domain
  source: Source
  timestamp: string
  read: boolean
  actionRequired: boolean
}

interface QueryResult {
  question: string
  answer: string
  sources: { type: Source; count: number }[]
  confidence: number
  relatedQuestions: string[]
}
```

## סיכום

המערכת הזו היא **הצעד הבא** אחרי ה-Insight Hub. היא הופכת את התובנות לכלי עבודה יומי — עם דוחות מותאמים, ניתוח עמוק, שאילתות חופשיות, והתרעות חכמות שמגיעות לאדם הנכון בזמן הנכון.

> **המטרה:** שאף מנהל לא יופתע. שכל כשל יזוהה לפני שהוא הופך למשבר. שכל החלטה תתבסס על דאטה אמיתי — לא על תחושת בטן.
